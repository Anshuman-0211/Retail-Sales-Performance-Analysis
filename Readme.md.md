# **Retail Sales Performance Analysis (SQL Project)**



### **Project Overview**

###### This project analyzes retail sales data using PostgreSQL to uncover insights related to sales performance, customer behavior, and product trends. The objective is to support data-driven decision-making through structured SQL analysis.



### **Dataset**

###### The dataset contains 1,000 retail transactions with attributes such as transaction date, product category, quantity sold, price per unit, customer demographics, and total transaction value.



### **Tools \& Technologies**

* ###### PostgreSQL
* ###### pgAdmin
* ###### SQL
* ###### GitHub



### **Business Questions Answered**

1. ###### What is the total revenue generated?
2. ###### How does revenue vary month over month?
3. ###### Which product categories generate the most revenue?
4. ###### What is the average transaction value?
5. ###### Which customer demographics contribute the most to revenue?
6. ###### Which transactions are above the average order value?



### **Key Insights**

* ###### Revenue is unevenly distributed across product categories.
* ###### Certain age groups and genders contribute more significantly to total revenue.
* ###### High-value transactions form a small but impactful portion of overall sales.
